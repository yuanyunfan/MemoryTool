## Architecture Notes

- **Shared SQLite**: GUI and MCP server are separate processes sharing one .db file via WAL mode
- **MCP Transport**: stdio only. Claude Code spawns MemoryMCP as subprocess
- **No print() in MCP target**: stdout is MCP protocol channel, use os_log/stderr for debugging
- **Swift MCP SDK**: official modelcontextprotocol/swift-sdk 0.12.0
- **DB Library**: GRDB.swift for type-safe SQLite access with FTS5 support
- **CJK Search**: trigram tokenizer + LIKE fallback for short terms, OR semantics for multi-word
- **Embedding**: NLContextualEmbedding (system BERT), mean pooling, L2 normalized
- **Vector Search**: Accelerate vDSP brute-force cosine similarity (sufficient for <10K memories)
- **Dedup**: SHA-256 hash on normalized content + cosine similarity > 0.92

## Database Schema (V3)
- memory: id, content, category, source, created_at, updated_at, metadata, content_hash, access_count, last_accessed_at, embedding
- memory_fts: FTS5 trigram (content, category, source)
- tag: id, name
- memory_tag: memory_id, tag_id
